1. 흥미로운도입
당신은 커피숍에 앉아 커피를 마시며 휴대폰으로 커뮤니티 게시판을 확인합니다.
여러 게시글 중 흥미로운 글을 발견하고 댓글을 발견합니다. 하지만 문득, '이 게시판이 어떻게 동작할까?'
라는 궁금증이 생깁니다. 게시글 CRUD 처리같은 기능은 어떻게 구현될까요? 이제 우리는 Spring 을 사용하여
이러한 게시판 기능을 구현하는 방법을 배워볼것입니다.

2. 학습목표
- Spring 프레임워크를 사요하여 게시판의 기본 CRUD 기능을 구현하는 방법을 이해합니다.
- 게시글 수정, 삭제 및 답글 기능의 구현방법을 배웁니다.
- 데이터베이스 와의 연동을 통해 게시판 데이터를 관리하는 방법을 학습합니다.

3. 주요핵심데이터와 개념
- CRUD
- Spring MVC : Spring 모델 뷰 컨트롤러 패턴을 사용하여 어플리캐이션을 구성
- JPA/Hibernate : Java에서 제공하는 ORM(Object-Relational Mapping)기술
	개게와 데이터벱이스의 테이블을 매핑합니다.
- Thymeleaf : Spring과 함께 용되는 서버가이드 Java탬플릿 엔진

4. 생각토론
- 게시판에서 CRUD기능 외에 사용자 경험을 향상시킬수 있는 추가기능은 무엇이 있을까요?
- 데이터베이스 설계에서 게시글과 답글 관계를 어떻게 표현하는 것이 효율적일까요? 
- 게시판의 보안을 강화하기위해 어떤 조치들이 필요한가요?

6. 실무예시
- 게시판구현 : 회사내부 네트워크에서 직원들이 사용할 수 잇는 커뮤니케이션 게시판을 구축하빈다.
	게시판 CRUD기능으 포함하며 답글 기능을 통해 커뮤니케이션을 촉진합니다.
- 데이터베이스 연동 : mySql또는 postgreSQL을 사용하여 게시판데이터를 관리합니다.
- 보안구현 : 게시글과 댓글에 대한 입력 데이터 검증으 실시하, XSS및 SQL 인젝션공격을
	방지하기 위한 보안조치를 적용합니다.

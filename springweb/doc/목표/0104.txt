### 1. 흥미로운 도입
당신은 큰 쇼핑몰의 매니저입니다. 쇼핑몰에서는 손님들이 자신의 쇼핑 카트에 물건을 담고, 또한 가게에서 일어난 이야기들을 기록합니다.
여기서, 세션 처리는 각 손님의 쇼핑 카드를 관리하고, 파일 업로드는 그들의 이야기를 기록하는 것과 같습니다. 
이제 Spring과 MyBatis를 사용하여 일한 기능을 구현하는 방법을 배워볼 것 입니다.

### 2. 학습목표
- Spring 기반의 웹 애플리케이션에서 세션 관리 방법을 이해합니다.
- MyBatis와의 연동을 통해 데이터베이스 작업을 처리하는 방법을 배웁니다.
- 파일 업로드를 위한 Spring MVC 구성 요소의 사영법을 학습합니다, 

### 3. 주요 핵심 키워드와 개념
- Spring Framework : Java 기반의 강력한 앤터프라이즈 애플리케이션 개발 프레임워크
- MyBatis : SQL 매핑 프레임워크로 복잡한 JDBC코드 작성 없이 데이터베이스와의 연동을 지원
- 세션 : 서버 측에서 사요자의 사태 정보를 유지하는 방법
- Multipart File : 웹에서 파일을 업로드할 떄 사용되는 데이터 형식
- Spring MVC : 모델(Model) 뷰(View) 컨트롤러(Controller) 아키택처를 기반으로 한 Spring 웹 프레임워크

### 4. 생각하고 토론해볼 내용 
- 세션 기반 인증과 토큰 기반 인증의 차이점과 장단점은 무엇일까요? 
- 대용향 파일 업로드 시 발생할 ㅜ 있는 문제와 그 해결방법은 무엇일까요?
- Spring 에서 Mybatis 를 사용하는 이점은 무엇이며? JPA와 비교했을때의 차이점은 무엇일까요?

### 5. 간단한 퀴즈
1. Spring Framework 에서 세션 정보는 어디에 저장됩니까?
2. Mybatis를 사용할때 Sql 쿼리는 어디에 정의 합니까?
3. 파일 업로드흘 위한 Spring MVC 의 어노테이션은 무엇입니까?

### 6. 실무적용 예시
- 세션관리 : 사용자 로그인 정보관리. 사용자가 로그인하면 해당 세션에 사용자 정보를 저장하고 
	로그아웃 시 세션을 만료시킵니다.
- 파일업로드 : 사용자 프로필 사진 업데이트. 사용자가 프로필 사진을 업로드 하면 서버에 저장하고 데이터 베이스에 
	파일 경로를 기록합니다.
